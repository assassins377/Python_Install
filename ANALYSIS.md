## 🔴 Критичные баги — ✅ ВСЕ ИСПРАВЛЕНЫ (2026-05-31)

### 1. `dispatch.py` — NameError: config ✅ исправлено
Использовался `config.INSTALL_LOGS_DIR`, но `import config` отсутствовал
(импортировались только `os, subprocess, wx, core, i18n, state, stats`).
Срабатывало когда упало > 1 программы и юзер жал «Да» — открыть папку логов.
Падало на обеих платформах.
**Сделано:** добавлен `import config` (`dispatch.py:8`).

### 2. `installer.py` (+ `updater.py`) — загрузки уходили не туда на Linux ✅ исправлено
```python
os.environ.get("TEMP", config.SCRIPT_DIR)   # было
```
На Linux нет переменной `TEMP` → скачанные инсталляторы падали в каталог
приложения вместо `/tmp`.
**Сделано:** `tempfile.gettempdir()` в `installer.py:398` и `updater.py:185`,
добавлен `import tempfile` в оба модуля.

### 3. `installer.py` — мёртвый код + Windows-логика в ветке скачивания ✅ исправлено
Выражения `shlex.join(...)` вычислялись и результат выбрасывался — остаток
рефакторинга. Дальше команда жёстко собиралась через `msiexec`/`regedit` —
для скачанного `.deb`/`.sh`/`.AppImage` на Linux не работало.
**Сделано:** диспетчеризация по расширению вынесена из `build_cmd` в
`core.dispatch_cmd(script_path, user_args)`; ветка скачивания делегирует туда же
(~25 строк мёртвого кода удалено). Путь больше не гоняется через строку —
избегаем поломки `shlex` на путях с пробелами.

**Проверки:** `py_compile` всех модулей OK; smoke-тест `dispatch_cmd`
(`.exe`/`.deb`→pkexec/`.sh`→bash) и `build_cmd` (bare `winget`) — корректно.
⚠️ Полный `pytest` не прогнан — в окружении нет `pytest`. Прогнать у себя:
`python -m pytest tests/ -q`.
